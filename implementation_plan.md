### Phase 1: Feature Engineering

#### [NEW] [features.py](file:///wsl.localhost/Ubuntu/home/darsh_run/DS_Final/sentinel_ds/src/features.py)

- Attempts `cudf.pandas` GPU acceleration; falls back to CPU Pandas
- `preprocess_features(df_chunk) â†’ X, y` with:
  - **Binary Label Creation**: Maps the `Label` column (e.g., "Benign", "Bot", "DDoS") to a binary `is_threat` target (0 for Benign, 1 for Malicious).
  - **Standard Scaling**: Dynamically scales numerical network flow features (`Flow Duration`, `Total Fwd Packets`, etc.) using `.partial_fit()`.
  - **Explicit Optimization**: Bypasses expensive stateless Z-Score/Rolling window generation and relies purely on the 80+ existing statistical features from CICFlowMeter.
  - **Feature Selection / Dimensionality Reduction**: Selects the most important predictive flow features to minimize memory footprint and latency.
  - **Data Type Casting**: Downcasts float64 to float32 to save memory and optimize GPU transfer.

### Phase 4: Orchestration

#### [NEW] [main.py](file:///wsl.localhost/Ubuntu/home/darsh_run/DS_Final/sentinel_ds/main.py)

1. Connect to CIC-IDS2018 data via `data_loader.py` `iter_batches()` to grab sequential chunks.
2. Temporal Training Phase (~80% of chunks chronologically):
   - Preprocess through `features.py`.
   - Update `ThreatDetector` via `.partial_train()`.
3. Online Simulation Phase (~20% of chunks chronologically):
   - Derives `threat_score` continuous probabilities.
   - Simulation loop over the test chunks:
     - `analyst_load` changes from 0.1â†’0.9 every $M$ events (5 phases)
     - Bandit decides action based on `[threat_score, analyst_load]`
     - Ground truth reward computed and fed back to update Bandit Q-table immediately.
7. Log per-step: `threat_score`, `action`, `true_label`, `reward`, `analyst_load`. Expose the live state (events, running metrics, log feeds, and active threats) via a FastAPI backend for the React dashboard to consume.
8. Plot 3 charts with `matplotlib` and save to `sentinel_ds/output/`:
   - **Recall & Precision** over simulation steps (rolling window)
   - **Action distribution** per analyst-load phase
   - **Cumulative reward** over time
